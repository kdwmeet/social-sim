# Generative Agents Simulation (대규모 사회 시뮬레이션)

## 1. 프로젝트 개요

이 프로젝트는 단일 목표를 수행하는 기존의 에이전트 구조를 넘어, 각기 다른 페르소나와 기억을 가진 여러 에이전트가 가상의 환경(World)에서 자율적으로 생활하고 상호작용하는 대규모 사회 시뮬레이션 시스템입니다.

시간(Tick)이 흐름에 따라 에이전트들은 자신의 성격과 과거 경험을 바탕으로 다음에 이동할 장소와 행동을 스스로 계획합니다. 만약 두 명 이상의 에이전트가 같은 장소에 모이게 되면 시스템은 자율적인 대화를 발생시키며, 이 대화 내용은 다시 각 에이전트의 장기 기억에 누적되어 향후 그들의 행동과 관계 형성에 영향을 미칩니다.

## 2. 시스템 아키텍처 및 워크플로우

본 시스템은 월드 전체의 상태를 중앙에서 관리하며, 에이전트들의 개별 행동과 군중 상호작용을 순차적으로 처리합니다.

1. State Definition: 월드의 현재 시간(time), 각 에이전트의 상태 및 기억(agents 딕셔너리), 그리고 세계에서 일어난 모든 사건을 기록하는 통합 관제 로그(global_log)를 전역 상태로 관리합니다.
2. Planning Node: 현재 시간과 각 에이전트의 페르소나, 과거 기억을 종합하여 이번 시간에 에이전트가 어디로 이동하여 무엇을 할지 구조화된 데이터(Pydantic)로 계획합니다.
3. Interaction Node: Planning 결과 동일한 장소(예: 카페, 공원)에 위치하게 된 에이전트들을 그룹화합니다. 모인 에이전트들 간의 자연스러운 대화를 창작하고, 이를 전체 로그와 개별 에이전트의 기억(Memory)에 각각 업데이트한 뒤 시간을 1시간 증가시킵니다.

## 3. 기술 스택

* Language: Python 3.10+
* Package Manager: uv
* LLM: OpenAI gpt-5-mini
* Data Validation: Pydantic (v2) (다수 에이전트의 행동 및 대화 데이터 구조화)
* Orchestration: LangGraph (다중 에이전트 행동-상호작용 순환 파이프라인), LangChain
* Web Framework: Streamlit (시간 진행 컨트롤 및 통합 관제 대시보드)

## 5. 핵심 준수 사항 (개발 가이드라인)

대규모 데이터를 다루는 본 시스템의 안정성을 위해 다음 규칙을 엄격히 적용했습니다.

* 상태 참조 안정성: 상태 딕셔너리 접근 시 필연적으로 발생할 수 있는 KeyError를 방지하기 위해 반드시 `state.get("키", 기본값)` 방식을 사용합니다.
* 예외 처리: Streamlit에서 `app_graph.stream()` 결과를 순회하여 UI를 렌더링할 때, 반환되는 상태 업데이트 값(`state_update`)이 None일 경우를 방어하는 Null Check(`if not state_update: continue`) 로직을 이중으로 구성했습니다.
* 프롬프트 템플릿 보호: ChatPromptTemplate 구성 시 파이썬 f-string 사용을 전면 배제했습니다. 복잡한 에이전트 컨텍스트나 로그 텍스트 내에 포함될 수 있는 특수 기호가 LangChain 템플릿 파서와 충돌하는 것을 원천 차단하기 위해, 변수는 체인 호출(`.invoke()`) 단계에서 딕셔너리로 안전하게 치환합니다.

uv venv
uv pip install -r requirements.txt
uv run streamlit run main.py

## 7. 테스트 시나리오 및 검증 방법

1. 시간 진행 및 행동 계획: '다음 시간으로 진행' 버튼을 눌러 시간이 8시에서 9시로 넘어갈 때, 각 에이전트가 자신의 직업이나 성격에 맞는 장소로 이동하는지 우측 글로벌 로그에서 확인합니다.
2. 공간 기반 상호작용: 두 명 이상의 에이전트가 우연히 같은 장소(예: 카페)에 도착했을 때, 시스템이 이를 감지하고 자연스러운 대화 로그를 생성해 내는지 점검합니다.
3. 기억의 영속성: 에이전트 간의 대화가 발생한 직후, 좌측의 에이전트 상세 상태 창을 열어 해당 대화 내용이 각 에이전트의 '기억 목록'에 올바르게 추가되었는지 확인합니다.
